# 第1章 CSS和文档

1. 层叠样式表中的“层叠”是指样式可以结合起来使用，而且具有优先级

2. 置换元素和非置换元素

   置换元素：用来置换元素内容的部分不由文档内容直接表示 img input

   非置换元素：大部分元素都是非置换元素

3. 块级元素和行内元素

   块级元素：默认生成一个填满父级元素内容区的框

   + 块级元素不能出现在行内元素中
   + 列表是特殊的块级元素
   + 置换元素可以是块级元素，但往往不是
   + 块级元素嵌套行内元素，之后通过CSS调换元素之间的嵌套关系，变成行内元素包裹块级元素是有效的

4. CSS样式表的分类

   + 行内样式表 权重最高 通常不建议使用
   + 内部样式表
   + 外部样式表

5. 外部样式表的链接方式

   1. link    ```<link rel="stylesheet" type="text/css" href="">```
      + 一个文档可以关联多个外部样式表，浏览器会加载指定的样式表，合并后应用到文档上
   2. import    ```@import url(./css/sheet.css)```
      + 当加载多个外部样式表的时候，浏览器会顺序应用外部样式表到文档上
      + 如果一个外部样式表需要用到另一个外部样式表中的样式，import的作用就提现出来了
      + 外部样式表中不能包含任何文档标记，不能使用link，但是可以使用import

# 第3章 特指度与层叠

1. 确定应该把哪些值应用到元素上时，用户代理不仅要考虑继承，还要考虑声明的特指度，以及声明的来源，在这个过程称之为层叠

2. 选择符的特指度由选择符本身的组成部分决定， 一个特指度值由四部分组成，0,0,0,0

   + ID：0,1,0,0        类：0,0,1,0		 选择符中的每个元素和伪元素：0,0,0,1
   + 伪类到底有没有特指度，CSS2中自相矛盾，但是CSS2.1中指明伪元素有特指度
   + 连结符合通用选择符不增加特指度

3. 特指度的应用过程

   在**任何情况下**，用户代理都会确定哪些规则与元素匹配，然后找出所有相关的声明，计算各自的特指度，判断哪些规则胜出，再把胜出的规则应用到元素上，得到装饰后的结果。

   每个元素、选择符和声明都要经历这一系列的操作。

   用户代理能够自动处理一切，这个行为是层叠的重要组成部分。

4. 通用选择符的特指度：通用选择符不增加特指度，特指度为0,0,0,0，这与没有特指度是不同的

5. 连结符没有特指度，因此，连结符对选择符的总特指度没有影响

6. 行内样式声明的特指度比其他任何声明的特指度都高

7. 重要声明

   + !important
   + 位置必须正确
   + 带有!important的声明对特指度没有影响，但会与不重要的声明区分处理，所有带有!important的声明会放在一起，特指度冲突就会在这个范围内解决。非重要的声明作为一个整体，其中的冲突使用特指度解决，因此，重要声明和非重要声明冲突时，重要声明始终胜出

### 继承

1. 属性值绝不会沿着树状图向上传播，但是有例外，应用到body元素上的背景样式会传给html元素，html元素是文档的根元素，用于定义渲染文档的画布。这一例外仅发生在为body元素定义了背景而没有为html元素定义背景的情况下
2. **继承的属性值没有特指度**
3. 滥用通用选择符的危害：通用选择符能匹配任何元素，它往往会终结继承，解决方法就是尽量不用通用选择符

### 层叠

1. 按权重和来源（创作人员、读者、用户代理）排序
2. 按特指度排序
3. 按前后位置排序
4. CSS的表现提示**（没看懂）P119**

# 第4章 值和单位

### 关键字、字符串和其他文本值

1. 关键字
   + 全局关键字：inherit、initial、unset
     1. inherit强制继承
        + 通常，直接指定的属性值能够覆盖继承的属性之，但是inherit能够将这种情况反过来
        + 缺点：样式融合太深，容易引起可访问性方面的问题
     2. initial重置
        + 把属性值设置为预定义的初始值
        + 不是所有的属性都有默认值，比如color，color的属性值取决于用户代理，color设置为initial的意思是：默认值取决于浏览器等的偏好设置
     3. unset：上面两个属性的通用替身
        + 对于继承的属性来说，作用与inherit一致
        + 对于不继承的属性来说，作用与initial一致
2. 字符串
   + 换行符的转义 \A
3. URL：绝对路径和相对路径
4. 图像
5. 标识符
